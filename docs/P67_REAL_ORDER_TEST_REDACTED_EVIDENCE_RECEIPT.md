# P67 Real `/order/test` Redacted Evidence Receipt

P67 receives a redacted result generated by the separately installed operator-side sender after one approved Binance Futures testnet `POST /fapi/v1/order/test` execution. It validates the P66 activation chain, exact testnet endpoint scope, operator request ID, metadata-only credential reference, key fingerprint, one-shot nonce, request/query/response/no-secret hashes, external-process signing and HTTP evidence, HTTP 200 response class, no-order-created truth, and timestamp bounds.

P67 never reads credentials, signs requests, sends HTTP, invokes the sender executable, consumes the nonce, calls an endpoint, submits an order, writes P7 status, mutates runtime, or promotes a stage.

## Important boundary correction

`/fapi/v1/order/test` validates request construction and authentication but does not create an order. Therefore a successful P67 receipt is **not**:

- a signed-testnet order submission,
- a post-submit execution record,
- a fill or position record,
- reconciliation evidence,
- session-close evidence,
- P50 external-submit evidence,
- P7 post-submit evidence.

P67 keeps `p50_external_evidence_import_eligible=false` and `p7_post_submit_evidence_import_eligible=false`. An accepted actual P67 receipt may only make the system eligible for a **separately approved signed-testnet submit preflight**.

Generated artifacts are templates and fixtures only. They do not prove that a real `/order/test` call occurred.
